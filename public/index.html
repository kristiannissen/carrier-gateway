How It Works

1. Book a Shipment

Send a single API request to book a shipment with any carrier. Support for single parcels, colli, COD, DDP, DAP, and receiver-pays options. 

POST /api/v1/shipments
Content-Type: application/json

{
  "carrier": "your_carrier",
  "service": "standard",
  "from": {
    "name": "Sender Inc.",
    "address": "123 Main St, Copenhagen"
  },
  "to": {
    "name": "Receiver A/S",
    "address": "456 Side St, Stockholm"
  },
  "parcels": [
    {
      "weight": 2.5,
      "dimensions": {
        "length": 30,
        "width": 20,
        "height": 10
      }
    }
  ]
}

2. Track in Real-Time

Get real-time tracking updates from all carriers through a single endpoint. No more polling multiple APIs. 

GET /api/v1/shipments/:id/tracking

{
  "status": "in_transit",

Built for Developers

First-Class Developer Experience

Our API is designed with developers in mind. Clear documentation, consistent naming conventions (Schema.org), and support for legacy formats like EDI.

  • RESTful endpoints for all functions (booking, tracking, service points)
  • JSON for all inputs/outputs
  • Open-sourced under Apache 2.0
  • Demo deployment on Vercel

Example: Book a Shipment

import "logistics-gateway/client"

func main() {
  client := client.New("https://api.logistics-gateway.dev")
  
  shipment := client.BookShipment("your_carrier", map[string]interface{}{
    "service": "standard",
    "from": map[string]string{
      "name": "Sender Inc.",
      "address": "123 Main St, Copenhagen",
    },
    "to": map[string]string{
      "name": "Receiver A/S",
      "address": "456 Side St, Stockholm",
    },
    "parcels": []map[string]interface{}{
      {
        "weight": 2.5,
        "dimensions": map[string]float64{
          "length": 30,
          "width": 20,
          "height": 10,
        },
      },
    },
  })
  
  fmt.Println("Tracking ID:", shipment.TrackingID)
}
